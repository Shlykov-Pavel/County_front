# Бэкенд — Северное графство

HTTP API на Go. Используется только стандартная библиотека: `net/http` с роутером из Go 1.22+ (паттерны вида `GET /api/health`), `log/slog` для логов. Без сторонних фреймворков.

## Требования

- Go 1.22+

## Запуск

```bash
make run                 # go run ./cmd/server
# или
make build && ./bin/server
```

Проверка:

```bash
curl localhost:8080/api/health
# {"status":"ok","time":"..."}
```

## Конфигурация

Через переменные окружения (см. `.env.example`):

| Переменная | По умолчанию | Описание |
|------------|--------------|----------|
| `PORT`     | `8080`       | Порт HTTP-сервера |
| `APP_ENV`  | `development`| Окружение |

## Структура

```
backend/
├── cmd/server/         # точка входа (main): конфиг, запуск, graceful shutdown
├── internal/
│   ├── config/         # загрузка настроек из окружения
│   └── httpapi/        # HTTP-слой: роутер, middleware, обработчики
│       ├── router.go   # регистрация маршрутов
│       ├── middleware.go # логирование запросов
│       └── health.go   # GET /api/health
├── go.mod
└── Makefile
```

## Команды Makefile

| Команда | Действие |
|---------|----------|
| `make run`   | Запустить сервер |
| `make build` | Собрать бинарник в `bin/server` |
| `make test`  | Прогнать тесты |
| `make vet`   | Статический анализ |
| `make tidy`  | Привести зависимости в порядок |
